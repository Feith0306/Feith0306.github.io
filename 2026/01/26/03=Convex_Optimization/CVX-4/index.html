凸优化学习系列 - 4. 梯度下降法

目标：读完这篇博客，应当能

  • 理解梯度下降法的基本原理与推导；
  • 学会如何用梯度下降法求解凸优化问题；
  • 理解梯度下降对步长选择的敏感性及收敛性；

一、梯度下降的问题具体形式

1. 无约束光滑优化（主线）

梯度下降最标准的讨论对象是无约束、可微（通常还要求光滑）的优化问题： \[ \begin{align} \min_{x\in\mathbb{R}^n}~ f(x), \end{align} \] 其中 \(f\) 为凸函数（凸优化场景）或一般非凸函数（非凸场景），并假设 \(f\) 可微。

  • 为何从无约束开始：无约束时更新形式最简洁，核心困难集中在“一阶信息 + 步长 + 曲率控制”。此框架也便于引出 Lipschitz 梯度、强凸、线搜索与收敛速率等基本概念。

2. 带约束凸优化：梯度下降并非不能用，但需改造

若存在凸约束 \(x\in\mathcal{C}\)（闭凸集），问题为： \[ \begin{align} \min_{x\in\mathcal{C}}~ f(x). \end{align} \] 此时“直接的” \(x^+=x-t\nabla f(x)\) 可能离开可行域，需要结合约束结构采用等价/扩展的一阶方法。

(A) 投影梯度法（Projected Gradient Descent, PGD）

有待后续文档中详细讨论 当集合 \(\mathcal{C}\) 的欧氏投影易计算时，使用： \[ \begin{align} x_{k+1} = \Pi_{\mathcal{C}}\!\left(x_k - t_k \nabla f(x_k)\right), \quad \Pi_{\mathcal{C}}(z):=\arg\min_{y\in\mathcal{C}}\|y-z\|_2. \end{align} \]

  • 适用约束类型：盒约束、球约束、单纯形、一些简单线性约束等（投影可高效实现）。
  • 要点：更新由“梯度步 + 投影回可行域”组成，本质上是将无约束梯度步的点通过最近点映射拉回约束集。

(B) 近端梯度法（Proximal Gradient）统一约束与不可微正则

有待后续文档中详细讨论 考虑复合形式： \[ \begin{align} \min_x~ f(x)+g(x), \end{align} \] 其中 \(f\) 可微凸，\(g\) 凸但可不可微。定义近端算子： \[ \begin{align} \operatorname{prox}_{t ,g}(z) :=\arg\min_y\left(g(y)+\frac{1}{2t}\|y-z\|_2^2\right). \end{align} \] 近端梯度更新为： \[ \begin{align} x_{k+1}=\operatorname{prox}_{t_k g}\left(x_k-t_k\nabla f(x_k)\right). \end{align} \]

  • 约束作为指标函数：令 \(g=\iota_{\mathcal{C}}\)，其中 \(\iota_{\mathcal{C}}(x)=0\) 若 \(x\in\mathcal{C}\)，否则 \(+\infty\)，则 \(\operatorname{prox}_{t \iota_{\mathcal{C}}}\) 恰为 \(\Pi_{\mathcal{C}}\)，因此 PGD 是近端梯度的特例。

(C) 约束复杂时：原始-对偶/障碍/增广拉格朗日等更常用（后续文档讨论）

当投影或近端子问题难以求解（例如一般不等式约束的复杂可行域），通常转向原始-对偶方法、内点法或增广拉格朗日法等，而非“朴素梯度 + 硬投影”。

二、梯度下降法的推导与解读

1. 为什么要做泰勒展开（或局部模型）且只到二阶？

梯度下降的经典推导来自在点 \(x\) 附近构造一个可控且易最小化的局部二次模型。对可微函数，\(f\) 在 \(x\) 附近的一阶近似为： \[ \begin{align} f(y)\approx f(x)+\nabla f(x)^\top (y-x). \end{align} \] 但仅用一阶项不足以“约束步子大小”，会导致线性模型在远处失真。为控制更新幅度，引入二次正则项（亦可视为“信赖域/近端项”）： \[ \begin{align} m_t(y;x)=f(x)+\nabla f(x)^\top (y-x)+\frac{1}{2t}\|y-x\|_2^2. \end{align} \]

  • 二阶的角色：该二次项不是在严格意义上“必须做二阶泰勒展开”，而是构造一个强凸的 surrogate（替代）模型，以确保子问题可解且更新不会过大。
  • 为何不展开更高阶：更高阶模型会显著增加每步代价；一阶方法的目标是保留“每步便宜”的优势，并以可控步长换取全局收敛保证。

2. 为什么用 \(\frac{1}{t}I\) 代替 Hessian？

若函数二阶可导，局部二阶信息由 Hessian \(\nabla^2 f(x)\) 决定。梯度下降可被理解为将 Hessian 近似为各向同性矩阵： \[ \begin{align} \nabla^2 f(x)\approx \frac{1}{t}I, \end{align} \] 从而避免显式计算/求解 Hessian 相关的线性方程（这是牛顿法的核心代价）。

  • 解释：\(\frac{1}{t}I\) 以“统一曲率尺度”替代真实曲率；\(t\) 越小，对步长惩罚越强，更新越保守。

3. 为什么更新律是 \(x^+=x-t\nabla f(x)\)？

令下一点 \(x^+\) 为上述二次模型的最小点： \[ \begin{align} x^+ = \arg\min_{y}\left(f(x)+\nabla f(x)^\top (y-x)+\frac{1}{2t}\|y-x\|_2^2\right). \end{align} \] 对 \(y\) 求导并令其为零（该问题对 \(y\) 强凸）： \[ \begin{align} 0 &= \nabla_y m_t(y;x) = \nabla f(x) +\frac{1}{t}(y-x),\\ \Rightarrow\quad y &= x-t\nabla f(x). \end{align} \] 因此得到梯度下降更新： \[ \begin{align} x^+=x-t\nabla f(x). \end{align} \]

4. argmin 形式与显式更新是否等价？

由上一推导可知： \[ \begin{align} x^+ = \arg\min_y\left(f(x)+\nabla f(x)^\top (y-x)+\frac{1}{2t}\|y-x\|_2^2\right) \quad\Longleftrightarrow\quad x^+=x-t\nabla f(x). \end{align} \] 二者完全等价，只是“优化子问题视角”与“闭式解视角”的两种写法。

5. 最优解是否唯一？不唯一时 GD 收敛到哪个解？

  • 若 \(f\) 强凸（或严格凸），则最优解唯一。
  • 若仅为凸但非严格凸，最优解集合 \(\mathcal{X}^\star\) 可为一个凸集，解不唯一。
  • 此时梯度下降通常收敛到某个 \(x^\star\in\mathcal{X}^\star\)，但具体收敛到哪一个最优解由初值与算法细节决定，一般无法在不加额外规则（例如正则化、约束偏好）的情况下指定。

三、如何选择更新步长

步长 \(t\) 决定每次更新幅度与稳定性。其核心作用可概括为：在局部线性化可信的范围内取尽可能大的步长，以加快收敛，同时避免过冲导致发散。

1. 为什么步长大了会发散、步长小了收敛慢？

用二次函数揭示本质。令： \[ \begin{align} f(x)=\frac{1}{2}x^\top Qx,\quad Q\succeq 0, \end{align} \] 则 \[ \begin{align} \nabla f(x)=Qx,\quad x_{k+1}=x_k-tQx_k=(I-tQ)x_k. \end{align} \] 收敛要求谱半径 \(\rho(I-tQ)<1\)，等价于 \[ \begin{align} 0<t<\frac{2}{\lambda_{\max}(Q)}. \end{align} \]

  • 步长过大：沿最大曲率方向出现 \(|1-t\lambda_{\max}|>1\)，误差被放大，迭代发散（典型“过冲 + 放大”）。
  • 步长过小：收缩因子接近 1，每次只减少很小比例，导致达到精度需大量迭代，表现为收敛慢。

2. 固定步长

若已知 \(L\)（Lipschitz 梯度常数），常用选择为： \[ \begin{align} 0<t\le \frac{1}{L}. \end{align} \] 优点是实现简单；缺点是 \(L\) 往往未知，且过保守时会显著减慢收敛。

3. Backtracking line search（回溯线搜索）

当 \(L\) 不可得时，backtracking 通过“充分下降条件”自适应选择步长。给定参数 \(0<\beta<1\)、\(0<\alpha<\frac{1}{2}\)，从初始步长 \(t=t_{\mathrm{init}}\) 开始，检查 Armijo 条件： \[ \begin{align} f(x-t\nabla f(x)) \le f(x)-\alpha t\|\nabla f(x)\|_2^2. \end{align} \] 若不满足，则缩小： \[ \begin{align} t \leftarrow \beta t, \end{align} \] 重复直到条件满足，然后更新 \(x^+=x-t\nabla f(x)\)。

(1) 该判断条件的物理意义

\(-\alpha t\|\nabla f(x)\|^2\) 可视为一次近似预测下降量 \(-t\|\nabla f(x)\|^2\) 的一个比例。该条件要求：真实函数值的下降至少达到线性模型所预测下降的一部分，即更新必须“足够有效”。

(2) 为什么不满足就缩步长？为什么不直接更新？

不满足 Armijo 条件说明步子太大导致线性化不可信，可能出现 \(f(x^+) \ge f(x)\) 或下降不足，从而破坏单调下降及后续收敛分析。缩小步长使更新落入“线性化可信”的范围，保证可以得到足够下降，再执行更新。

(3) 为什么取 \(0<\beta<1\) 与 \(0<\alpha<1/2\)？

  • \(0<\beta<1\)：保证步长按几何级数缩小，有限次缩小即可进入可接受区间，从而保证线搜索终止。
  • \(0<\alpha<1/2\)：在 \(L\)-光滑条件下，可证明存在足够小的步长区间使 Armijo 条件成立；取 \(\alpha<1/2\) 使该区间不至于过窄，并避免过度保守（经验上也较稳健）。

(4) 为什么 backtracking 能保证收敛“速度”？

backtracking 不改变理论收敛阶（一般凸仍为 \(O(1/k)\)），但它能避免两类灾难： - 步长过大导致发散； - 步长过小导致极慢。 在 \(L\)-光滑条件下，可证明 backtracking 选出的步长最终与 \(1/L\) 同量级，因此在常数因子意义上接近“合理固定步长”的效率。

4. Exact line search（精确线搜索）

Exact line search 沿负梯度方向求一维精确最小： \[ \begin{align} t=\arg\min_{s\ge 0} f(x-s\nabla f(x)). \end{align} \] 区别与评价：

  • exact line search 需要解一维优化子问题，通常要多次函数评估甚至导数评估，单步代价大；
  • backtracking 只需验证“充分下降”，通常少量试探即可得到可接受步长；
  • 在许多问题中，梯度法慢的主因是方向与条件数（几何结构导致“之字形”），而非步长是否最精确。因此 exact 的额外开销往往不能换来同等量级的迭代减少，整体效率可能不如 backtracking。

四、收敛性分析

1. Convergence rate（收敛速率） 与 \(O(\cdot)\) 的定义及意义

(1) 收敛速率（convergence rate）

用误差序列刻画迭代的收敛速度。常见误差包括： \[ \begin{align} \text{函数值误差: }~ f(x^k)-f^\star, \qquad \text{驻点误差: }~ \|\nabla f(x^k)\|. \end{align} \]

例如，\(f(x_k) - f(x^\star) \le \frac{C}{k}\) 表示函数值误差以 \(O(1/k)\) 速率收敛。

(2) 大 \(O\) 记号

对序列 \(\{a_k\},\{b_k\}\)，若存在常数 \(C>0\) 与 \(k_0\)，使得对所有 \(k\ge k_0\)： \[ \begin{align} a_k \le C\, b_k, \end{align} \] 则记为 \(a_k = O(b_k)\)。

2. 四个定理（凸/强凸/下界/非凸）及关键概念解读

定理 1：光滑凸（convex + differentiable + Lipschitz gradient）

条件：

  • \(f\) 凸且可微；
  • \(\nabla f\) 为 \(L\)-Lipschitz： \[ \begin{align} \|\nabla f(x)-\nabla f(y)\|_2 \le L\|x-y\|_2,\quad \forall x,y. \end{align} \]
  • 若 \(f\) 二阶可导，则上述条件等价于： \[ \begin{align} \nabla^2 f(x) \preceq LI,\quad \forall x, \end{align} \] 即 Hessian 的最大特征值不超过 \(L\)（曲率上界）。

结论（固定步长）： 当 \(0<t\le 1/L\)， \[ \begin{align} f(x^{(k)})-f^\star \le \frac{\|x^{(0)}-x^\star\|_2^2}{2tk}. \end{align} \] 因此函数值误差为 \(O(1/k)\)，达到精度 \(\varepsilon\) 的迭代复杂度为 \(O(1/\varepsilon)\)。

解释： 仅有曲率上界，没有曲率下界时，目标函数可在某些方向“非常平”，无法保证每步按固定比例压缩误差，故只能得到亚线性速率。

定理 2：光滑强凸（strongly convex + differentiable + Lipschitz gradient）

条件：

  • \(f\) 为 \(m\)-强凸且可微；
  • \(\nabla f\) 为 \(L\)-Lipschitz。
  • 若二阶可导，则强凸等价于： \[ \begin{align} \nabla^2 f(x) \succeq mI,\quad \forall x, \end{align} \] 即 Hessian 的最小特征值不小于 \(m\)（曲率下界）。

结论（固定步长）： 当 \[ \begin{align} 0<t\le \frac{2}{m+L}, \end{align} \] 存在 \(0<\gamma<1\) 使得 \[ \begin{align} f(x^{(k)})-f^\star \le \gamma^k\cdot \frac{L}{2}\|x^{(0)}-x^\star\|_2^2. \end{align} \] 因此为线性（几何）收敛；常写迭代复杂度为 \[ \begin{align} k = O\!\left(\frac{L}{m}\log\frac{1}{\varepsilon}\right). \end{align} \]

解释： 当 \(mI\preceq \nabla^2 f \preceq LI\)，函数像“均匀碗形”，每一步都能保证按固定比例缩小误差；但比例由条件数 \(\kappa=L/m\) 控制，\(\kappa\) 越大，收敛越慢。

定理 3（Nesterov 最坏情形下界）：一阶方法不可能比 \(O(1/k^2)\) 更快

问题背景： 对 \(L\)-光滑凸函数类，考虑“一阶方法”——每次迭代点 \(x^{(k)}\) 仅能依赖于已访问的梯度信息，并位于由历史梯度张成的线性子空间中。

结论（下界形式）： 当 \(k\le (n-1)/2\) 时，对任意一阶方法，存在某个 \(L\)-光滑凸函数使得 \[ \begin{align} f(x^{(k)})-f^\star \ge \frac{3L\|x^{(0)}-x^\star\|_2^2}{32(k+1)^2}. \end{align} \]

理解： 该定理说明：在最坏情形下，任何一阶方法对光滑凸问题的可保证速率不可能优于 \(O(1/k^2)\)。这同时解释了两点：

  • 普通梯度下降的 \(O(1/k)\) 在最坏情形下不是最优阶；
  • 但存在加速一阶法能达到 \(O(1/k^2)\)，且该阶在该函数类上已经是最优（不能再提高阶数）。

定理 4：非凸光滑——全局最优通常不可求，转向近似驻点（\(\varepsilon\)-substationary）

为什么“asking for optimality is too much”？ 非凸优化可能具有大量局部极小、鞍点与复杂地形；寻找全局最优在一般情形下往往是计算上极其困难的问题（常需全局搜索/组合爆炸），因此“要求全局最优”在算法保证层面通常不现实。

非凸问题是否存在最优解？ 可能存在也可能不存在：若 \(f\) 只有下确界 \(\inf f\) 但不取到该值，则不存在全局最小点。即使存在，全局最小点的个数与结构一般也难以仅凭局部信息确定。

梯度下降在非凸情形收敛到什么？ 典型保证是收敛到“驻点”意义的解（梯度接近 0），而非全局最优或必然局部最小。可能停在： - 局部最小点； - 鞍点附近； - 或其他满足一阶必要条件的点。

\(\varepsilon\)-substationary point 的定义： 点 \(x\) 若满足 \[ \begin{align} \|\nabla f(x)\|_2 \le \varepsilon, \end{align} \] 则称为 \(\varepsilon\)-substationary（近似驻点）。

关键结论（非凸收敛指标）： 在 \(L\)-光滑下，取 \(0<t\le 1/L\)，可得到 \[ \begin{align} \min_{0\le i\le k}\|\nabla f(x^{(i)})\|_2 \le \sqrt{\frac{2\left(f(x^{(0)})-f^\star\right)}{t(k+1)}}. \end{align} \] 因此要获得 \(\|\nabla f\|\le \varepsilon\)，需要 \[ \begin{align} k = O(1/\varepsilon^2). \end{align} \]

五、梯度下降求解凸优化

5.1 伪代码

算法1：固定步长梯度下降

\[ \begin{array}{l} \textbf{Algorithm 1: Fixed-Step Size Gradient Descent}\\ \textbf{Input:} \ x^{(0)},\ \alpha>0, \ K,\ \varepsilon>0\\ 1:\ \textbf{for } k=0,1,\dots,K-1\ \textbf{do}\\ 2:\ \ \ \ g^{(k)} \leftarrow \nabla f\!\left(x^{(k)}\right)\\ 3:\ \ \ \ \textbf{if }\ \|g^{(k)}\|\le \varepsilon\ \textbf{then break}\\ 4:\ \ \ \ x^{(k+1)} \leftarrow x^{(k)}-\alpha\, g^{(k)}\\ 5:\ \textbf{end for}\\ \textbf{Output:} \ (x^{(k)},\ f\!\left(x^{(k)}\right)) \end{array} \]

算法2：变步长梯度下降（Backtracking Line Search）

\[ \begin{array}{l} \textbf{Algorithm 2: Gradient Descent with Backtracking Line Search }\\ \textbf{Input:} x^{(0)},\ \alpha_0>0, \ \beta\in(0,1),\ c\in(0,1),\ K,\ \varepsilon>0\\ 1:\ \textbf{for } k=0,1,\dots,K-1\ \textbf{do}\\ 2:\ \ \ \ g^{(k)} \leftarrow \nabla f\!\left(x^{(k)}\right)\\ 3:\ \ \ \ \textbf{if }\ \|g^{(k)}\|\le \varepsilon\ \textbf{then break}\\ 4:\ \ \ \ \alpha \leftarrow \alpha_0\\ 5:\ \ \ \ \textbf{while }\ f\!\left(x^{(k)}-\alpha g^{(k)}\right) > f\!\left(x^{(k)}\right) - c\,\alpha\,\|g^{(k)}\|^2\ \textbf{do}\\ 6:\ \ \ \ \ \ \alpha \leftarrow \beta\,\alpha\\ 7:\ \ \ \ \textbf{end while}\\ 8:\ \ \ \ x^{(k+1)} \leftarrow x^{(k)}-\alpha\, g^{(k)}\\ 9:\ \textbf{end for}\\ \textbf{Output:} \ (x^{(k)},\ f\!\left(x^{(k)}\right)) \end{array} \]

  • backtracking line search 不改变梯度法在一般光滑凸上的收敛阶（仍为 \(O(1/k)\)），但能通过自适应步长避免发散与过慢；
